# AIOS

AIOS is a personal AI operating system initiative focused on building a local-first, knowledge-centric, governed cognitive infrastructure.

This repository is a combined **architecture + future implementation monorepo**. It is intentionally being bootstrapped architecture-first so implementation can evolve from a clear, durable, and traceable foundation.

## Vision

Build a local-first Personal AI Operating System whose primary assets are governed knowledge, traceable decisions, and bounded AI workflows.

AIOS is not intended to be merely a chatbot application. It is intended to become durable cognitive infrastructure for thinking, learning, planning, software development, research, and execution over many years.

## Guiding priorities

1. Architecture before implementation
2. Knowledge before agents
3. Governance before autonomy
4. Long-term maintainability over short-term productivity

## Repository approach

This repository combines strategic architecture artifacts and future platform implementation in one monorepo, while keeping strong separation between:

- architecture and governance artifacts
- knowledge and ontology artifacts
- platform and runtime implementation
- workflows and agent-role definitions
- experiments and exploratory work

## Repository structure

- `docs/` — overview, glossary, roadmap, and repository guidance
- `architecture/` — principles, capability map, target architecture, standards
- `governance/` — governance model, traceability, autonomy, policy-oriented artifacts
- `knowledge/` — knowledge architecture, taxonomy, and information management approach
- `ontology/` — minimal viable ontology and later machine-readable evolution
- `adr/` — architecture decision records
- `platform/` — future implementation for infrastructure and platform components
- `workflows/` — workflow definitions and operational playbooks
- `agents/` — agent roles, policies, and evaluation artifacts
- `experiments/` — bounded experiments that do not define the platform by default

## Core architecture stance

AIOS should be:

- local-first where practical
- model-agnostic
- knowledge-centric
- traceable end-to-end
- incrementally autonomous
- simple in foundations and explicit in governance
